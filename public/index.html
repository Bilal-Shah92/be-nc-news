GET /api/topics

This endpoint returns a list of all the topics. Each topic returns a slug and a description.

Acceptable Queries

This endpoint does not accept any other queries.

Format of request body

This endpoint does not expect a request body.

Example Response

{
  "topics": [
    {
      "slug": "coding",
      "description": "Code is love, code is life"
    },
    {
      "slug": "football",
      "description": "FOOTIE!"
    },
    {
      "slug": "cooking",
      "description": "Hey good looking, what you got cooking?"
    }
  ]
}

GET /api/articles

This endpoint returns a list of all the articles. You can choose to sort them by any valid column, choose ascending or descending order and filter them by topic.

Acceptable Queries

This endpoint accepts 3 query options.

You can query by ?sort_by= which allows you to sort the articles by columns such as /api/articles?sort_by=votes. This means sort the articles by votes.
If column is not included then it defaults to "created_at"

You can query by ?order= which decides the order of your sorting such as /api/articles?order=asc.
If no order is included then it defaults to descending order.

You can query by topic such as /api/articles?topic=coding, this will only show articles relating to the topic of coding.
If no topic is included, then all articles are returned.

Format of request body

This endpoint does not expect a request body.
